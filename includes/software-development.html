WARTA app Game Launcher // 2020-2021

Fully implemented the game launcher for The Last Frontier VR project. Designed UI (via Figma) of the application, thought out the user behavior, then implemented the logic and user interaction using C# and WPF (Visual Studio, Blend for Visual Studio, Git).

Since the project The Last Frontier VR is a rather specific project (vending attraction), the launcher has several specific and different functions compared to ordinary PC launchers, namely, setting up the calibration of the attraction sensors for different game maps.

The launcher has a self-updating module and support for several versions: stable and beta (for testing experimental features in the production stage)

Implemented downloading and updating the game client. There is support for choosing which version of the game to download: stable or beta version (to be able to download the beta-client, you need to enter the secret keyword in a special field)

In the implementation of the launcher, the MVVM design pattern is used. There is a logging of critical errors in a separate file. You can enable a visual display of the leaderboard in the Main window. Implemented the function of automatic detection of the COM port of the plug-in custom controller and the choice of the sound output channel for the Buttkicker (one of the key features of the attraction)

Language module allows to easily add different localizations for the application (currently supported languages: Russian and English). Localization texts are stored in separate files and can be easily edited.

Tools and plugins

Warta App Manifest Builder

Implemented a mini-app for generating a Manifest file. The manifest file is needed to determine the list of files that need to be downloaded when updating the game client. The manifest file stores data in JSON format.

This file is stored on the server and the launcher accesses it to get the current version of the game and the list of files.

Unity Custom Inspector

During the development of projects, quite often there is a need for additional convenience in working with Inspector. The project has implemented non-standard Inspector (with Gizmos visualization) for some massive Plugins and Components (for example, Player Controller and Controller for AI, Enemy components and Shooting system).

Unity Editor Extensions and Tools

Implemented custom systems and additional tools (localization system, pooling system, parameters saving system, developer console), new UI components (tab group, flexable grid layout, animated banners, etc.)
